# Frontend Coding challenge - E-commerce product page

![Design preview for the E-commerce product page coding challenge](./design//product-page/desktop-design.jpg)

**To do this challenge, you need a good understanding of HTML, CSS and JavaScript.**

## ⚔️ The challenge

Your challenge is to build out this e-commerce product page and get it looking as close to the design as possible.

Your users should be able to:

- View the optimal layout for the site depending on their device's screen size
- See hover states for all interactive elements on the page
- Open a lightbox gallery by clicking on the large product image
- Switch the large product image by clicking on the small thumbnail images
- Add items to the cart
- View the cart and remove items from it
- View the cart items even if the page is reloaded or changed to another web URL.

## 🔍 Where to find everything

Your task is to build out the project to the designs inside the `/design` folder. You will find both a mobile and a desktop version of the design.

The designs are in JPG static format. Using JPGs will mean that you'll need to use your best judgment for styles such as `font-size`, `padding` and `margin`.

You will find all the required assets in the `/images` folder. The assets are already optimized.

There is also a `style-guide.md` file containing the information you'll need, such as color palette and fonts.

We have also include another small guide that will gonna help you to create the Home page: [Home page guide](./design/home-page/required-note-guide.md)

## 🏗️ Building your project

1. This folder is also your repository, you can start coding inside of it right way.
2. Initialize your project as a public repository on [GitHub](https://github.com/). Creating a repo will make it easier to share your code.
3. Look through the designs to start planning out how you'll tackle the project. This step is crucial to help you think ahead for CSS classes to create reusable styles.
4. Use what you have learned before from your previous projects by including SCSS and respect the clean code way.
5. You need to structure well your JS code by splitting it in multiple files ex: (main.js, product.js, cart.js, etc...).
6. Product API URL is: `./api/products.json`, simply just fetch it to get the needed data. You don't need any external package to do this job.
7. Do not even think of using AI. Force yourself to code it alone but you have the full access to google what you have miss or what you didn't understand.
8. Do not try to edit the `product.json` file or rename images or files.
9. Finally, this is the most important part: Push commits progressively on GitHub. Those commits must clearly reflect your development process, with meaningful commit messages that document each significant change or milestone.

## 🔦 Guidance

- There is an bug in the design the quantity input that's inside the product page should not have 0. The minimum quantity should be 1.
- The checkout button action should refresh the page since we don't actually have a checkout page.
